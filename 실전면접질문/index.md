## 4. GET 방식, POST 방식에 대해 설명해주세요

GET과 POST는 서버를 통해 데이터를 전달하는 방식입니다.
GET은 보통 URL 주소 뒤에 parameter를 붙여서 전달하며 값이 직접적으로 보이므로 URL을 책갈피로 사용, form 기능과 별도로 사용 가능합니다. 한편 값이 보이므로 보안 취약이나 너무 긴 길이의 값은 원활한 처리가 힘들 수 있습니다.
POST는 HTTP Request header에 포함시켜 전달하며 GET방식보다 보안성이 높으며 별도의 길이제한이 없습니다.

## 12. 다음 이미지 포맷에 대해 각 특징에 대해 설명 해 주세요. (JPG, GIF, PNG, SVG)

JPG는 손상 가능성이 있지만 용량을 크게 줄일 수 있어 일반적으로 많이 사용됩니다. 다만 투명 처리가 힘든 단점이 있습니다.<br>

> PNG는 화질도 좋고 투명 처리도 가능한 포맷으로 알고 있습니다.<br>
> SVG는 벡터들을 조합해 만든 이미지로 크기를 아무리 줄이거나 늘여도 화질이 깨지지 않습니다. 그러나 복잡한 이미지라면 용량을 많이 차지하게 됩니다.

## 16. 다음 코드 중 1번과 2번의 차이점을 설명해주세요.

```js
function Car(){};
1. var car  = Car();
2. var car = new Car();
```

각각 일반 함수, 생성자 함수로 호출되었습니다. 1번은 Car()라는 함수의 return 값을 변수에 담는 코드이고, 2번은 생성자를 통해 Car()라는 인스턴스를 변수에 담는 코드입니다.

> 1번은 Car 함수가 일반 함수로 호출이 되었고, 2번에서는 Car 함수가 생성자 함수로서 호출이 되었습니다.<br>
> 일반 함수로 호출된 함수는 해당 함수 내부의 return 값을 반환하여 1번의 car 변수에는 undefined가 할당되고,<br>
> 생성자 함수로서 호출된 함수는 인스턴스를 반환하여 2번의 car 변수에는 Car 객체가 할당됩니다.
